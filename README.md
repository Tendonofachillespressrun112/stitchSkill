## 🚀 What stitchSkill does

stitchSkill helps you turn a product brief into a wireframe plan and then into screens in Google Stitch. It works with Antigravity, so the app can read your document, map the full flow, and build a clear screen list before it starts drawing.

This is useful when you want:

- A full set of wireframes from one document
- Consistent navigation across screens
- Less hand work when making mockups
- A cleaner way to move from text to layout

## 📄 How to use it

### 1. Load your document

Add the file that describes your app, site, or workflow. This can be a product brief, feature list, or screen outline.

### 2. Review the screen plan

The app creates a list of screens based on the document. Check the order and names so they match your project.

### 3. Generate the context prompt

stitchSkill builds a prompt with the full flow in mind. This gives Stitch more context for each screen.

### 4. Send the plan to Google Stitch

Use the generated prompt with Stitch through Antigravity. The app keeps the flow linked, so one screen leads to the next.

### 5. Review the wireframes

Open the result in Stitch and check the layout, labels, and navigation.

### 6. Make changes if needed

If a screen looks wrong, update the source brief or screen plan and run the process again.

## 🧠 How it works

stitchSkill follows a simple path:

1. Read the document
2. Find the main screens
3. Build a full screen map
4. Create context for each step
5. Pass that context to Stitch
6. Generate wireframes with a linked flow

This helps avoid the common issue where each screen feels separate. The app keeps the full product in view, so the flow stays close to the source document.

## 🔧 Common file types

stitchSkill works best with simple source files that describe the product in plain text. Good examples include:

- `.pdf`
- `.docx`
- `.txt`
- `.md`

If your brief is long, keep the sections clear. Use headings for pages, actions, and user steps. That helps the app build a better screen list.

## ⚙️ Tips for better results

- Use one document for one app or feature set
- Name screens in a clear way
- List all main actions on each page
- Keep navigation rules in the brief
- Mention repeat elements like headers, sidebars, and footers
- Add user roles if the app has more than one type of user

A clear brief gives you a better wireframe plan.

## 🖥️ Troubleshooting

### The app does not open

- Check that the file finished downloading
- Right-click the file and choose Run as administrator
- Make sure Windows did not block the file
- Move the file to a simple folder like Downloads or Desktop

### The wireframes look incomplete

- Use a clearer brief
- Add more page names
- Describe the main user flow
- Separate one big brief into smaller parts

### Stitch does not match the flow

- Check the screen order in the plan
- Make sure the brief mentions shared navigation
- Add more detail about which page comes next

### The app cannot read the file

- Try another file type
- Save the brief as `.txt` or `.md`
- Remove special formatting if the document came from another app
